# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Monorepo structure

pnpm workspaces + Turborepo. All apps under `apps/`, shared packages under `packages/`.

- `apps/backend` ‚Äî FastAPI + PostgreSQL + MinIO. See `apps/backend/CLAUDE.md` for full detail.
- `apps/frontend` ‚Äî React 19 + TanStack Router + React Query + Tailwind CSS 4 + shadcn/ui
- `packages/api` ‚Äî Auto-generated React Query + Axios client (Orval from FastAPI OpenAPI spec)

## Commands

```bash
# Workspace
pnpm install
pnpm turbo dev                              # Run all apps in parallel

# Frontend (Vite ‚Üí http://localhost:3001)
pnpm -F frontend dev
pnpm -F frontend build                      # tsc + vite build
pnpm -F frontend lint

# Backend (http://localhost:8000/docs)
cd apps/backend
docker compose up -d                        # PostgreSQL :5433 + MinIO :9000
pip install -r requirements.txt
uvicorn main:app --reload

# DB migrations (Alembic)
alembic revision --autogenerate -m "description"
alembic upgrade head

# API client codegen (backend must be running)
pnpm orval
```

No test infrastructure exists yet (no pytest, no vitest).

## Frontend architecture (`apps/frontend`)

### Routing

TanStack Router with file-based routing. Route files live in `src/apps/` (not `src/routes/`). `src/routeTree.gen.ts` is auto-generated by the Vite plugin ‚Äî never edit it manually. Adding a new route = new file in `src/apps/`.

Route files in `src/apps/` are thin wrappers ‚Äî they just import and render from `src/pages/<feature>/`. All page logic (React Query calls, state, composables) lives in `src/pages/`.

`src/apps/_protected/route.tsx` is the auth guard. All authenticated pages nest under `_protected/`.

### Provider stack (order matters)

Entry point: `src/providers/AppProvider.tsx`.

```
QueryClientProvider
  ‚îî‚îÄ AuthProvider          ‚Üê JWT auth, role/permission resolution
       ‚îî‚îÄ CityProvider     ‚Üê selected city state (persisted in localStorage)
            ‚îî‚îÄ MapProvider ‚Üê MapLibre GL instance + light preset + 3D terrain state
                 ‚îî‚îÄ RouteWithContext  ‚Üê RouterProvider with auth context injected
```

`RouteWithContext` passes auth context into TanStack Router via `createRootRouteWithContext`.

### Auth flow

`AuthProvider` uses `useReducer` with states: `BOOT` ‚Üí `RESTORING` ‚Üí `AUTHENTICATED` | `UNAUTHENTICATED`.

On mount, it executes in sequence:
1. `currentUserUsersMeGet()` ‚Äî validates JWT cookie, gets user
2. `getUserRolesUserRolesUserIdGet(userId)` ‚Äî resolves role
3. Permission set resolved via hard-coded `ROLE_PERMISSIONS` map in `AuthProvider.tsx` (mirrors backend seed)
4. City access loaded by role: `investor` calls `myAccessCityAccessMeGet()`, `lgu_admin` calls `listAssignmentsLguAssignmentsGet()`

`signIn()` / `register()` set cookie + dispatch `AUTH_SUCCESS`, then navigate to `/city-setup`. `register()` accepts an optional `role` param (`'investor' | 'lgu_admin'`, defaults to `'investor'`). `signOut()` clears cookie + navigates to `/login`. `refreshCities()` re-resolves city access without signing out ‚Äî call after granting/revoking city access.

### Permission & role hooks (`src/hooks/use-permission.ts`)

- `usePermission(perm: string) ‚Üí boolean` ‚Äî checks if current user has permission
- `useRole() ‚Üí string | null` ‚Äî returns current user's role slug
- `useCityIds() ‚Üí string[]` ‚Äî returns city IDs the user can access

Use the `PERMISSION` constants object from `src/config/permissions.ts` instead of raw strings:

```ts
import { PERMISSION } from '@/config/permissions'
usePermission(PERMISSION.HAZARD_READ)
```

### Geo contexts

`CityProvider` tracks the currently selected city. Selection is persisted to `localStorage` under key `biznest:selected_city` and validated against `city_ids` from auth on restore ‚Äî cleared if the city is no longer accessible or on sign-out. Consumed via `useCityContext()`.

`MapProvider` holds the `MapEngine` instance, light preset, and 3D toggle state. When `selectedCity` changes, `MapProvider` fetches the city boundary and calls `engine.flyToCityBoundary()` + `engine.setCityBoundary()` automatically. Consumed via `useMapContext()`.

### MapEngine (`src/engine/map.engine.ts`)

`MapEngine` is a chainable wrapper around MapLibre GL JS. All map operations (sources, layers, markers, popups, events, terrain) go through it ‚Äî never call `maplibregl` methods directly in components.

Key patterns:
- All `add*` methods are idempotent (check before adding source/layer)
- All `remove*` methods are safe (no-op if not present)
- `addPmtilesSource(id, url)` ‚Äî registers a `pmtiles://` vector source; PMTiles protocol is auto-registered at module load
- `setCityBoundary(boundary)` ‚Äî renders outside-city dark overlay + boundary outline; `clearCityBoundary()` removes both
- `isInsideBoundary(lng, lat)` ‚Äî point-in-polygon check using ray-casting (returns `true` when no boundary set)
- `setTerrain(enabled)` ‚Äî adds MapTiler DEM source and enables 3D terrain; requires `VITE_MAPTILER_KEY`
- `addImageOverlay(id, url, corners)` / `updateImageOverlay(id, corners)` / `removeImageOverlay(id)` ‚Äî raster image overlaid on the map using `ImageCorners` (4 `[lng, lat]` pairs: top-left, top-right, bottom-right, bottom-left)

Hardcoded PMTile source-layer names: NOAH hazard tiles use `slice`, faultlines use `faultline`.

### ZoningMap ‚Äî own MapLibre instance

`src/pages/zoning/zoning.map.tsx` creates its own `MapEngine` and overrides `MapContext` locally ‚Äî it does **not** use the `MapEngine` from `MapProvider`. This is intentional: the zoning view needs full map control without the hazard-loading side effects from `MapProvider`. It still reads `lightPreset` and `show3D` from the outer `MapProvider` context to stay in sync with user preferences.

The `useGeoreference` composable (`src/pages/zoning/composables/use-georeference.ts`) drives the image georeferencing feature: upload a raster image, position it on the map via draggable corners, adjust opacity/rotation, then save. It calls `engine.addImageOverlay` / `updateImageOverlay` / `removeImageOverlay`.

### Navigation config (`src/config/navigation.ts`)

Role-specific nav sections:
- `investorNavSections` ‚Äî investor sidebar (Dashboard, Cities, Map, Subscription, Saved Locations)
- `lguNavSections` ‚Äî LGU admin sidebar (Dashboard, Cities, Map, Alerts, Data + System group)
- `defaultNavSections` ‚Äî fallback for unrecognized roles
- `administrationNavSections` ‚Äî admin-only section (User Management, Role & Permissions); appended by `app-shell.tsx` for superusers

When adding a route that should appear in the sidebar, add it to the relevant section here ‚Äî don't hard-code nav items in `app-shell.tsx`.

### Data fetching

React Query via Orval-generated hooks in `packages/api/generated/<tag>/`. Each FastAPI router tag becomes its own folder. Axios is the HTTP client. Check `packages/api/index.ts` ‚Äî some exports may be commented out. Page components live in `src/pages/` organized by feature; complex pages extract logic into `composables/` subdirectories (e.g., `src/pages/city-setup/composables/use-city-setup.ts`).

Import from the `packages/api` package using the `@networking/api` path alias:
```ts
import { someHook } from '@networking/api/generated/cities/cities'
import type { CityResponse } from '@networking/api/model/cityResponse'
```

### UI

shadcn/ui (radix-nova style) in `src/components/ui/`. Path alias `@` ‚Üí `src/`. Main layout shell: `src/components/app-shell.tsx`.

## API client codegen

`orval.config.js` pulls from `http://localhost:8000/openapi.json`. Outputs React Query hooks split by OpenAPI tag into `packages/api/generated/<tag>/`, TypeScript schemas into `packages/api/model/`. No build step ‚Äî TypeScript exported directly. Run `pnpm orval` after any backend route/schema change.

## Backend summary

See `apps/backend/CLAUDE.md` for full architecture. Non-obvious points for frontend work:

- `HazardArea` is province-scoped (NOAH data); routes are under `/provinces/{province_id}/hazards/*` ‚Äî pass `province_id` directly, no city-level hazard filtering
- Hazard PMTile URLs are presigned MinIO URLs with **5-hour TTL** ‚Äî frontend must re-fetch when expired; do not cache them indefinitely
- `InvestorSubscriptionResponse.plan` is a nested `SubscriptionPlanResponse` object (`{ id, name, max_cities: number | null, created_at }`), not a string
- Route endpoints return `HazardAreaSummary` (no geometry) ‚Äî geometry lives only in PMTiles
- `get_db` session generator lives in `utils/jwtUtils.py`, not `core/db.py`
- `tippecanoe` (PMTile generation) only runs on Linux/Mac/WSL ‚Äî use `--skip-pmtiles` flag on Windows when seeding
